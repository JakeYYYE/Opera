## 一、运营平台模块（服务撮合与管理）

## 1. 背景与目标

- 背景
  - 低空经济相关应用（物流、测绘、农林植保、应急等）快速发展，但需求分散在各类零散渠道，服务商能力难以对接。
  - 现有平台要么偏招投标、要么只做展示，缺少一个轻量、规范、聚焦“找服务+发布需求”的中台。
  - 针对大规模的航拍爱好者、内容创作者和飞手群体，缺少一个“飞行地图”的工具型入口，既能帮助他们快速找到合规、安全、出片好的机位，又能自然将这部分高活跃流量沉淀到低空服务场景中。
- 产品目标
  - 面向地市级/行业用户，提供一站式低空服务需求发布、服务商展示与匹配的前台门户。
  - 为运维与运营同学提供统一的后台管理和数据总览能力。
  - 不做复杂的招投标业务，仅做“非招投标”的规范化需求管理与展示。

## 2. 范围与边界

- 本期范围（V1.0）

  - 前台
    - 首页（智慧飞滴入口、需求分类入口、优质服务商、新闻资讯）。
    - 需求大厅（列表+筛选+分页）。
    - 需求详情页（需求全量信息展示）。
    - 服务商库（服务商卡片、筛选、分页）。
    - 飞行地图（地图+点位基础展示，用于导流；详细需求见 DOC/飞行地图PRD.md）。
  - 后台
    - 后台总览（关键指标卡片+快捷入口）。
    - 需求管理（增删改查、筛选、分页、跳转前台详情）。
    - 服务商管理（增删改查、筛选、上下架、分页）。
    - 用户管理（C端用户认证/升级审核、B端商户管理）。
- 通用

  - 登录/注册入口（前端弹窗入口，后续可嫁接实际登录系统），按钮与弹窗遵循统一的品牌化渐变样式。
  - 注册角色选择：在注册弹窗中区分“我要发需求”和“我是服务商”，默认选择需求方，注册后通过“身份中心”可随时补充与切换身份。
  - 身份切换（MVP）：单账号多身份，无需重登；在“身份中心”切换活跃身份；权限由“活跃身份 + 已认证等级”决定入口与可见动作。
  - **[MVP 特别调整] 权限全开放**：当前阶段为验证业务流程，移除所有基于用户等级（Lv1-Lv4）和身份认证（个人/企业）的权限限制。任何登录用户默认拥有最高权限（Lv4 服务商 + 个人/企业需求方双重身份 + 已实名/已持证），可自由访问所有页面与功能。
  - 首页“后台管理”入口（从前台直接进入后台）。
- 不在本期范围内

  - 招投标全流程：招标公告、报名、竞标、开标、中标、公示、保证金等。
  - 金融结算：在线支付、发票、合同签署等。
  - 交易结算与托管：平台不参与撮合后的资金结算；不提供代收/代付、保证金托管。当前仅提供配额型工具服务（发布/聊天配额）。
  - 精细用户体系与权限：复杂多角色 ACL（本期后台权限暂按“管理员已登录”假设）。

## 3. 用户与角色

- 访客用户

  - 主要行为：浏览需求列表与详情、查看服务商信息、查看新闻资讯。
- 用户身份与切换（MVP策略，按账号端内切换）

  - 单账号可绑定多身份：需求方（个人/企业）与供给方（Lv1–Lv4）。
  - **账号端类型固定**：注册为 B 端企业账号的用户，其账号主体始终为企业；注册为 C 端个人账号的用户，其账号主体始终为个人，不支持在前端将企业账号“变更”为个人账号，反之亦然。
  - **同端内身份切换**：
    - B 端账号：可在「企业服务方」与「企业需求方」之间切换；若缺少目标身份能力，则进入对应的企业资质申请流程。
    - C 端账号：可在「个人服务方」与「个人需求方」之间切换；若缺少目标身份能力，则进入对应的个人认证/飞手申请流程。
    - 不允许 B 端账号申请成为 C 端身份，也不允许 C 端账号申请成为 B 端身份。
  - 身份切换在登录态内完成；在已有目标身份能力时，切换需要重新登录以保证权限刷新（弹出确认框后退出并回到登录入口）。
  - 权限由“活跃身份 + 已认证等级”决定可见入口与可执行动作。
  - 仅在改手机号/密码、异常风控、管理员强制下线时需重登。
- 需求方用户

  - 个人需求方：可发布个人级订单（婚庆、简单拍摄）。
  - 企业需求方：可发布企业级订单（巡检、测绘等）；前期可通过“企业咨询”表单，由后台代发。
- 供给方用户（飞手/服务商分层）

  - Lv1 认证飞手：实名认证与基础资料，浏览地图/社区。
  - Lv2 签约飞手：上传执照 + 保险声明 + 人工审核；可接个人级订单。
  - Lv3 服务商（小B）：上传营业执照 + 保证金声明 + 人工审核；可接企业级订单、入驻服务商库。
  - Lv4 金牌服务商：通过人工标记（MVP），获得推荐位与增值权益。
- 管理员

  - 维护需求与服务商；人工创建/分配订单（企业侧代发）；审核用户升级。

## 4. 核心流程

### 4.1 需求管理流程（MVP）

- 类型字段：`type = personal | enterprise`（个人级/企业级）。前台列表按类型分 Tab 展示。
- 配额：发布需求时扣减对应“发布配额”；撤回是否返还可在MVP阶段按运营策略配置。
- 个人需求流转：

  1. 个人需求方前台发布 → 轻审 → 上架“个人需求”大厅。
  2. Lv2 签约飞手“接单/锁定” → 执行 → 完工提交 → 评价。
- 企业需求流转：

  1. 企业需求方前台发布或后台代发 → 审核 → 上架“企业需求”大厅（或直接指派）。
  2. 运营手动分配给 Lv3/Lv4 服务商 → 执行 → 完工 → 评价。

### 4.2 服务商管理流程

1. 管理员在后台「服务商管理」中点击「新增服务商」。
2. 填写：名称、Logo、评分、成交量、标签、所在区域、上架状态。
3. 上架状态为“上架”时，在前台首页“优质服务商”和「服务商库」中参与展示。
4. 后续如需临时屏蔽某服务商，可通过「下架」操作，前台不再展示该服务商。
5. 无效或重复服务商，可通过「删除」彻底删除记录。

### 4.3 导航与入口流程

1. 访客打开前台首页 `/`。
2. 顶部导航展示：智慧飞滴 / 飞行地图 tab，导航菜单包含「首页」「个人中心」「消息」「后台管理」。
3. 点击「后台管理」，跳转到 `/admin`，自动重定向至后台总览 `/admin/dashboard`。
4. 后台侧边栏可在「总览 / 需求管理 / 服务商管理」间切换。

### 4.4 飞行地图流量激活流程

1. 用户入口
   - 来源场景：搜索“城市航拍/飞行地图”、朋友圈/公众号内容种草、线上宣推落地页等。
   - 用户在首页顶部 tab 中选择「飞行地图」，进入地图工具页。
2. 工具使用
   - 用户在地图上浏览城市/区域内可飞或适合拍摄的机位点位。
   - 可结合图层（如景观、地标、海拔等）筛选视角和构图，后续可叠加限飞区、空域信息。
3. 价值交付
   - 对飞手：快速解决“今天去哪飞、去哪拍”的决策问题，提高飞行频次和内容产出效率。
   - 对平台：沉淀高活跃、对低空场景敏感的用户流量，为后续需求发布和服务撮合埋下触点。
4. 转化导流
   - 在机位详情或侧栏推荐中，展示相关需求（如该区域的巡检/宣传拍摄需求）或推荐服务商。
   - 引导飞手/用户进一步访问「服务需求」「服务商库」模块，形成从“工具流量”到“业务流量”的自然转化。

  - 副标题：「可视化流程 · 规范化 · 提高效率 · 降低成本」。

  - 位置：首页主内容右侧（1/3 栅格）。
  - 顶部
    - 标题：「优质服务商」。
    - 描述：「品质保障」。
    - 右侧入口：「全部」跳转 `/service-providers`。

    - 展示若干服务商卡片（名称、标签、评分、成交量等）。

  - 首页展示最新资讯卡片（图片/标签/发布时间/标题/摘要），点击跳转详情；「查看更多」跳转列表页。

  - 路由：/news（列表）、/news/:id（详情）；与首页联动（卡片直达详情、查看更多至列表）。
  - 数据源：统一使用 useMockData.js 中的 news 数组，支持 localStorage 持久化与字段迁移。

### 5.2 需求大厅（/service-needs）

- 顶部导航
  - 与首页风格一致，tab 以「服务需求」为当前高亮。
- 结构
  - 双 Tab：个人需求 | 企业需求（按 `type` 切分）。
  - 权限控制：Lv2 仅显示“个人需求”；Lv3/Lv4 仅显示“企业需求”。
  - 发布入口：发布需求需扣减发布配额；无配额引导充值或企业咨询。
- 筛选条件（在各自 Tab 下）
  - 分类：多选或下拉。
  - 关键字：支持按标题、区域、分类进行模糊匹配。
  - 状态：进行中 / 已结束。
- 列表卡片字段
  - 标题、行业分类、区域、发布时间/日期、浏览量、预算、状态。
- 分页
  - 支持翻页组件，默认每页 10 条（前端分页模拟或后续接入后端）。

### 5.5 飞行地图（/find-spot）

- 产品定位

  - 作为面向飞手和内容创作者的工具型产品，解决“去哪飞、去哪拍”的决策问题。
  - 作为整个平台的流量激活入口，通过高频、轻决策的使用场景，持续带来自然流量。
- 当前形态（V1.0）

  - 地图视图 + 点位展示：
    - 使用地图组件展示城市/区域，已有若干示例点位。
    - 每个点位可展示基本信息（名称、简短说明），后续可补充机位照片等。
    - 推荐航拍点采用缩略图网格（3列）展示，支持悬停放大与标题遮罩。
  - 工具栏与交互：
    - 右侧垂直工具栏：创建航拍点、测距、测面积。
    - 创建航拍点：进入创建模式后鼠标样式切换，在地图上点击选点并上传/填写信息；支持**右键点击**取消创建模式并恢复默认状态；保存后出现在“我的航拍点”。
    - 测距：连续点选形成折线，鼠标移动显示临时线段；右键或双击结束；段距标签居中显示；“清点”移入测距面板。
    - 测面积：多边形点选，实时临时连线；完成后显示区域与中心面积标签；提供撤销/完成/清面操作。
  - 标签与弹层：
    - 左上标签改为“航拍点”，上下箭头提示当前状态。
    - 点击标签弹出内容浮层（推荐航拍点与我的航拍点），支持遮罩点击关闭。
  - 与首页导航打通：
    - 首页顶部 tab 直接切换到「飞行地图」。
    - 后台总览提供快捷入口，方便运维查看工具使用效果。
- 飞行地图的角色与价值

  - 对用户：
    - 帮助新手飞手熟悉城市可飞区域和典型机位，降低“怕违规/怕被劝离”的心理门槛。
    - 为内容创作者提供灵感，提升飞行频次和内容产出黏性。
  - 对平台：
    - 通过高频工具使用，将更多对低空场景有兴趣的用户聚合到平台内。
    - 可在合适时机引导用户查看相关需求或推荐服务商，形成闭环；地图内的推荐点位与用户航拍点以首张照片缩略图形式展示，点击后通过弹窗查看详情而非路由跳转。
- 后续可扩展方向

  - 叠加低空通道、限飞区、空域等级等图层，增强安全合规属性。
  - 增加“最佳机位”与“推荐时间段”（如日出、蓝调）等信息，强化拍摄指导。
  - 与需求/服务商建立关联（如某个点位由某服务商长期维护或运营活动）。

### 5.6 移动端适配与响应式体验（新模块）

- 模块目标
  - 在不单独开发 H5/小程序的前提下，通过响应式布局保证核心流程在手机端可顺畅使用。
  - 统一移动端交互模式：列表/表格 → 卡片、侧栏筛选 → 抽屉/顶部筛选、悬浮操作 → 底部/卡片操作区。
- 适配范围（前台）
  - 首页与飞行地图：已通过 Tailwind 栅格与断点类保证移动端展示不严重错位，本模块仅补充对核心交互页的要求。
  - 消息中心（/messages）：
    - 布局：桌面端保持左右双栏（消息列表 + 会话详情），移动端采用“列表视图 / 详情视图”双视图切换：
      - 默认展示消息列表；点击单条消息后进入详情视图。
      - 详情视图顶部提供“返回列表”入口，支持在会话与列表之间来回切换。
    - 气泡与输入区：
      - 文本气泡在移动端缩小左右边距与字号，确保一屏可呈现多条消息。
      - 报价方案（plan_proposal）以自适应宽度卡片展示，内部字段（报价金额、工期、说明等）在小屏不产生横向滚动。
      - 底部输入区预留安全区（safe area），在全面屏设备上避免被系统手势区域遮挡。
    - 列表增强：
      - 消息列表右侧增加未读红点标记（基于 `msg.unread` 字段），用于快速识别未读会话。
  - 个人中心相关列表：
    - 我的任务（ProviderTasks，供给方视角）：
      - 桌面端：保留 `el-table` 形式，字段包含任务名称、需求方、成交价、状态、截止日期与操作列。
      - 移动端：表格折叠为卡片列表：
        - 卡片头部展示任务标题与状态标签。
        - 中部展示“需求方、成交价、截止日期”等关键信息。
        - 底部操作区：
          - 当状态为“已通过”时，提供“提交成果 + 联系对方”按钮。
          - 其他状态提供“查看详情 + 联系对方”按钮，常用操作一屏可见。
    - 我的需求（RequesterNeeds，需求方视角）：
      - 桌面端：保留表格视图，字段包含需求标题、类型、预算、状态、发布时间及操作列。
      - 移动端：以卡片形式展示：
        - 顶部为标题 + 状态标签。
        - 中部一行用于展示需求类型标签与预算信息。
        - 底部展示发布时间，并在右侧提供“查看详情 / 撤销”操作按钮。
      - 发布入口：
        - 顶部“发布需求”按钮在移动端缩减为圆形图标按钮；桌面端则为带文案的大按钮，强调入口。
  - 服务商能力展示：
    - 我的服务能力（ProviderServices）：
      - 框架：继续采用服务卡片展示服务名称、描述、定价与上下架状态。
      - 移动端：
        - 顶部“发布新服务”入口使用圆形图标按钮形式，减少横向占用空间。
        - 卡片底部“编辑 / 上架/下架”操作在移动端默认展示；桌面端仍通过悬浮（hover）触发显示，以保持列表的干净简洁。
- 设计规范与技术要求
  - 布局能力：
    - 统一使用 Tailwind CSS 断点类（如 `md:hidden` / `hidden md:block`）控制不同屏幕下的组件显隐。
    - 表格类数据在移动端优先采用卡片形式，字段精简为“标题 + 2–3 个关键属性 + 状态 + 主操作”。
  - 交互模式：
    - 列表 → 详情类页面在移动端遵循“先列表后详情”的单列导航模式，并提供显式返回入口。
    - 关键操作（提交成果、撤销、联系对方等）在移动端应出现在卡片底部或底部操作条中，无需横向滚动即可触达。
  - 适配验收标准：
    - 在 375px 视口宽度（典型手机竖屏）下，相关页面不出现横向滚动条。
    - 卡片内主要字段在一屏内可阅读，重要操作按钮不被遮挡。
    - Navbar 和消息输入区在主流全面屏设备上不被系统手势区域遮挡。

## 7. 非功能需求

- 性能

  - 前台首页与列表页在正常网络环境下，首屏可交互时间 ≤ 2 秒（构建后本地环境参考）。
  - 构建后单个 JS 包超过 500 kB 时，仅作为警告，后续可通过拆包优化。
- 兼容性

  - 浏览器：Chromium 内核（Chrome、Edge 等）为优先支持目标。
  - 移动端：保证基础展示与操作不出现严重错位（主要布局为响应式 Tailwind）。
- 安全

  - 不在前端代码中硬编码任何密钥、Token 等敏感信息。
  - 后台入口仅做路由入口，实际权限控制可在后续通过登录态/网关完成。
- 可扩展性

  - 前后台数据目前以 mock 为主，后续可通过统一 API 层接入真实后端。
  - 需求与服务商数据结构设计为可扩展字段，方便增加更多属性。

## 10. 配额与计费（MVP）

- 模式：不做结算，仅售卖使用配额（credits）。
- 配额类型：
  - 发布配额：按个人/企业分别计；发布一次扣 1。
  - 聊天配额：对接会话扣减（每次发起或按每日限额）。
- 套餐示例：基础包（5次发布 + 20次聊天）、企业包（10次发布 + 50次聊天）。
- 展示与扣减：个人中心与发布入口展示剩余配额；发布/发起聊天时扣减。
- 续费：同一账号充值配额；不涉及收款/提现。
- 合规声明：平台不参与交易结算与发票；聊天仅用于对接，线下自行结算。

## 11. 飞行地图（工具型流量激活）KPI与度量

- 流量与使用
  - 日活 DAU：≥ 500（试运行阶段），后续逐步提升。
  - 机位浏览/会话：人均 ≥ 3 个点位。
  - 平均会话时长：≥ 90 秒。
- 转化与导流
  - 飞行地图 → 需求大厅跳转率：≥ 8%。
  - 飞行地图 → 服务商库跳转率：≥ 5%。
  - 收藏/标记机位的用户占比：≥ 10%（后续能力）。
- 质量与反馈
  - 机位有效性反馈（不合规/无法拍摄）响应时间：≤ 48 小时。
  - 新增机位覆盖：重点城市每月新增 ≥ 20 个优质机位。
